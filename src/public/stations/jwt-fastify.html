JWT Access + Rotating Refresh via @fastify/jwt

This station demonstrates the same JWT workflow as Station 8 (hand-rolled JWT), but uses @fastify/jwt instead of calling jsonwebtoken directly.

Key differences: two scoped plugin instances (fastify.access and fastify.refresh) handle separate secrets. request.accessVerify() reads and verifies the Authorization: Bearer header automatically. Signing and verification are synchronous (fast-jwt under the hood); refresh token rotation and revocation still use the in-memory store.
