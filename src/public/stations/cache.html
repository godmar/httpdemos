Cache-Control Station

This station shows how response cache headers change browser/proxy caching behavior.

Endpoints and Policies

GET /api/cache/public   -> Cache-Control: public, max-age=60
GET /api/cache/private  -> Cache-Control: private, max-age=30
GET /api/cache/no-store -> Cache-Control: no-store

What These Headers Mean

public, max-age=60: any cache (browser or shared proxy) may reuse this response for 60 seconds.

private, max-age=30: only a private cache (typically the browser) should cache it, for 30 seconds.

no-store: do not store the response at all in any cache.
